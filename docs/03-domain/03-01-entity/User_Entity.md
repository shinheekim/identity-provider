# User 엔티티

## 1. 개요
User는 시스템에서 사용자를 대표하는 최상위 엔티티이다.  
하나의 User는 여러 인증 수단(Identity)을 가질 수 있으며, 계정 상태와 기본 사용자 정보를 관리한다.

---

## 2. 테이블 정보

- 테이블명: `users`
- 설명: 사용자 기본 정보 및 계정 상태 관리

---

## 3. 필드 정의

### 3.1 기본 식별 정보

- **id**
    - User 고유 식별자
    - PK
    - 내부 시스템에서 사용하는 기본 키

- **userUuid**
    - UUID 기반의 고유 식별자
    - 외부 노출용 식별자로 사용할 수 있음
    - 생성 후 변경되지 않음

---

### 3.2 계정 상태

- **accountStatus**
    - 사용자 계정의 현재 상태를 나타냄
    - 계정 활성화 여부, 제한 여부, 탈퇴 여부 등을 관리하기 위한 상태값

상태 예시는 다음과 같다.

- `PENDING`: 가입 진행중
- `ACTIVE`: 정상 이용 가능
- `SUSPENDED`: 일시 이용 제한
- `BANNED`: 영구 이용 제한
- `WITHDRAWN`: 탈퇴
- `DORMANT`: 휴면

```java
public enum AccountStatus {
PENDING,    // 가입 진행중
ACTIVE,     // 정상 이용 가능
SUSPENDED,  // 일시 이용 제한
BANNED,     // 영구 이용 제한
WITHDRAWN,  // 탈퇴
DORMANT     // 휴먼
}
```


---

### 3.3 사용자 기본 정보

- **email**
    - 사용자 이메일 주소
    - 선택 입력 가능 
      - 현재는 필수 입력기능

- **emailVerified**
    - 이메일 인증 여부
    - 이메일이 인증되었는지 여부를 나타냄

- **phoneNumber**
    - 사용자 전화번호
    - 선택 입력 가능

- **phoneVerified**
    - 전화번호 인증 여부
    - 전화번호가 인증되었는지 여부를 나타냄

---

### 3.4 활동 정보

- **lastLoginAt**
    - 마지막 로그인 일시
    - 로그인 성공 시 갱신됨

---

### 3.5 공통 관리 정보

- **createdAt**
    - 생성 일시

- **updatedAt**
    - 수정 일시

---

## 4. 연관 관계

### User - Identity

- 하나의 User는 여러 Identity를 가질 수 있다
- Identity는 사용자의 로그인 수단을 의미한다

예를 들면 다음과 같은 형태가 가능하다.

- 일반 로그인용 Identity
- 카카오 로그인용 Identity
- 구글 로그인용 Identity

- User는 사람 또는 계정의 주체를 의미 
- Identity는 실제 인증 수단을 의미

---

## 5. 설계 의도

- User는 사용자 자체를 표현하는 엔티티로 사용한다
- 로그인 방식은 Identity로 분리하여 확장 가능하도록 설계한다
- UUID를 통해 외부 노출 시 내부 PK 노출을 방지할 수 있다
- 계정 상태를 별도로 관리하여 휴면, 정지, 탈퇴 등의 정책을 유연하게 처리할 수 있다

---

## 6. 고려 사항

- `accountStatus`는 문자열보다는 Enum 기반으로 관리하는 것이 안전하다
- `email`, `phoneNumber`의 중복 허용 여부는 정책적으로 추가 결정이 필요하다
- `WITHDRAWN` 상태의 데이터 보존 기간 및 삭제 정책이 필요하다
- `DORMANT` 전환 조건은 일정 기간 미로그인 기준으로 정의할 수 있다

---
