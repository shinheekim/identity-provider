# 프로젝트 소개 (Project Introduction)

## 1. 개요
본 프로젝트는 다양한 로그인 방식을 통합 관리하는 **인증 서버(Auth Server)** 를 구축하는 것을 목표로 한다.

기존 서비스마다 개별적으로 구현되어 있던 로그인/인증 로직을 분리하여,
독립적인 인증 시스템으로 구성함으로써 유지보수성과 확장성을 향상시키고자 한다.

---

## 2. 배경

현재 서비스 환경에서는 다음과 같은 문제점이 존재한다.

- 서비스마다 로그인 로직이 중복 구현됨
- 카카오, 구글 등 소셜 로그인 연동 시 일관성 부족
- 인증 정책(비밀번호, JWT 등)이 통일되어 있지 않음
- 보안 정책 변경 시 전체 서비스 수정 필요

이러한 문제를 해결하기 위해 인증 기능을 모듈화하고,
중앙 집중형 인증 서버를 도입한다.

---

## 3. 목표

### 3.1. 초기목표
초기 목표는 다음과 같다.
- 매번 새로운 서비스를 구현하는데 로그인에 대한 개발 시간을 줄이고 모듈화를 진행하도록 하여 새로운 서비스를 개발할때 해당 서버만 실행하면 로그인은 구축되도록 설계

### 3.2 주요목표 
본 프로젝트의 주요 목표는 다음과 같다.

- 로그인/인증 기능을 독립적인 서버로 분리
- 일반 로그인 + 소셜 로그인 통합 관리
- JWT 기반 인증 구조 설계 및 운영
- 인증 정책(비밀번호, 로그인 이력 등) 표준화
- 다양한 서비스에서 공통으로 사용할 수 있는 인증 플랫폼 구축

---

## 4. 주요 기능

- 일반 로그인 (ID / Password)
- 소셜 로그인 (카카오, 구글)
- JWT 발급 및 검증 (Access Token / Refresh Token)
- 로그인 이력 관리
- 비밀번호 정책 관리 (변경 주기, 이력 등)
- 계정 연동 (소셜 계정 연결)

---

## 5. 기대 효과

- 인증 로직의 중복 제거
- 서비스 간 일관된 로그인 경험 제공
- 보안 정책 중앙 관리 가능
- 신규 서비스 연동 시 개발 비용 감소
- 유지보수 및 확장성 향상

## 7. 대상 사용자

---

## 8. 용어 정의
- **Identity**: 사용자 식별 정보 (계정 단위)
- **JWT (Json Web Token)**: 인증 정보를 담은 토큰 기반 인증 방식
- **Access Token**: API 요청 시 인증에 사용되는 토큰
- **Refresh Token**: Access Token 재발급을 위한 토큰
- **Auth Server**: 인증을 담당하는 중앙 서버
- **Social Login**: 외부 플랫폼을 통한 로그인 (카카오, 구글)
